# Chrona 路线图

> **当前版本：** 0.1.2
> **状态：** 阶段一 — 排期驾驶舱

---

## 产品定位

Chrona 围绕三个核心产品区域演进：

- **日程创建与安排** — 将想法转化为结构化、有时间约束的工作
- **任务自动完成** — 让 AI 智能体在持续监督下执行工作
- **智能体集成** — 让 Chrona 成为智能体的"行动终点"，从对话、邮件、笔记中自动接收并结构化工作

长期目标：让排期不只是被动日历，而是**持续驱动真实任务执行的控制面**。

---

## 设计原则

1. **排期与执行不应该彼此脱节** — 排期的计划和智能体执行的计划是同一个
2. **计划应该既能在执行前编辑，也能在执行中更新** — 不是静态指令
3. **运行时选择应当是基础设施问题，而非用户需要承受的复杂度** — 一套 UX，多种后端
4. **即使自动化尚未完全成熟，产品本身也要先足够可用** — 每个成熟阶段都有实际价值

### 智能体集成 — MCP 工具 / Skill 🚧 *(开发中)*

> 目标：让 Chrona 成为 AI 智能体的"行动终点"。当智能体在对话中识别出待办工作，可以直接推入 Chrona 的执行管线。

Chrona 将暴露 MCP（模型上下文协议）工具或 Skill，供外部智能体（OpenClaw、Claude 等）调用，以程序化方式创建任务、生成计划并排期。这让每一次 AI 对话都成为结构化、可执行任务的来源。

| 能力 | 状态 |
|------|------|
| MCP 工具：从智能体上下文创建任务（标题、描述、优先级、时间块） | 🚧 开发中 |
| MCP 工具：为新任务或已有任务生成执行计划 | 📋 规划中 |
| 智能体主动询问用户："要不要把这个排进 Chrona？" | 📋 规划中 |
| 从外部来源（邮件、笔记等）通过智能体解析自动创建任务 | 📋 规划中 |
| Chrona 作为可注册 Skill 供 OpenClaw 及其他兼容运行时使用 | 📋 规划中 |

**关键用例：**

1. **对话 → 任务** — 你正在和智能体聊一个 bug；智能体识别到并主动询问：*"要不要我在 Chrona 里创建一个任务来跟踪这个修复？我帮你排到明天上午。"*

2. **邮件 → 任务** — 智能体读取你的收件箱，识别出待办事项，自动创建带优先级和时间块的 Chrona 任务——无需手动录入。

3. **会议纪 → 执行计划** — 智能体处理会议纪要，生成带子任务的 Chrona 结构化计划，并请求你的确认。
