# Chrona 路线图

## 产品定位

Chrona 正在围绕两个核心产品区域演进：
- 日程创建与安排
- 任务自动完成

长期目标不是做一个被动的日历，而是让“排期”直接成为驱动任务执行的主控制面。

## 路线图

### 阶段一：排期 cockpit

目标：
- 让日程创建更快、更智能
- 让排期页面真正可日常使用
- 减少从“想到一件事”到“安排成一段工作”的摩擦

关键能力：
- 创建任务或时间块时的智能提示
- 根据不完整输入生成任务建议
- 执行前的 AI 任务计划生成
- 对生成计划进行快速查看与调整

### 阶段二：具备计划意识的任务执行

目标：
- 把已经排期的工作与真实智能体执行连起来
- 让任务计划从静态说明变成可演化对象

关键能力：
- 根据日程自动触发智能体运行
- 通过配置好的运行时执行任务
- 让执行进度与任务计划保持联动
- 在执行推进过程中自动更新计划

### 阶段三：多运行时后端支持

目标：
- 在保持 Chrona 产品模型稳定的前提下，支持不同 AI 运行时

后端支持方向：
- 裸 LLM：适合轻量规划与执行流程
- OpenClaw：适合偏 agent 式执行流程
- Hermes：适合更深的智能体/工具编排

设计目标：
- 一个统一产品工作流
- 多种底层运行时实现
- 在不同运行时之间共享任务、排期与计划模型

## 设计原则

- 排期与执行不应该彼此脱节
- 计划应该既能在执行前编辑，也能在执行中更新
- 运行时选择应当是基础设施问题，而不是用户需要承受的复杂度
- 即使自动化尚未完全成熟，产品本身也要先足够可用
